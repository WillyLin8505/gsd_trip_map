## Phase Goal

**As a** trip planner, **I want to** fetch a resolved place's opening hours and a sensible default visit duration, with repeat lookups served from cache, **so that** the optimizer can schedule it later without re-billing Google.

<objective>
Deliver the cache-first Place Details slice: a GET /api/places/details endpoint that returns full place data (opening hours, coordinates, derived visit duration) and only calls Google on a cache miss or stale row, enforcing the 30-day TTL. Adds placeDetails() to the Google client with the Enterprise field mask (regularOpeningHours + utcOffsetMinutes), the hours_unknown rule (absent hours are NEVER treated as always open), and the visit-duration lookup table the Phase 2 optimizer needs.

Purpose: This is the cost-control core (Success Criterion 2 — cached second lookup) and the data the optimizer depends on. Aggressive caching is the only mitigation for the Enterprise SKU cost of opening hours.
Output: placeDetails(), durationForTypes(), GET /api/places/details with cache-first logic.
</objective>
