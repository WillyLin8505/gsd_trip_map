  <action>
    Create src/lib/db/migrations/0002_rls.sql enabling RLS and the policies from RESEARCH.md:

    places: ENABLE ROW LEVEL SECURITY; SELECT policy USING (true) — shared cache readable by all; INSERT and UPDATE policies restricted to auth.role() = 'service_role'. This keeps places writable only by the service-role server path while readable by everyone.

    itineraries: ENABLE ROW LEVEL SECURITY; an owner policy USING (auth.uid() = user_id) covering owner CRUD; plus a SELECT policy USING (is_public = true AND share_token IS NOT NULL) so public itineraries are readable via share token. The is_public flag must be enforced here server-side, not just hidden in the UI.

    itinerary_days: ENABLE ROW LEVEL SECURITY; owner policy USING (itinerary_id IN (SELECT id FROM itineraries WHERE user_id = auth.uid())).

    place_visits: ENABLE ROW LEVEL SECURITY; owner policy USING (itinerary_day_id IN (SELECT d.id FROM itinerary_days d JOIN itineraries i ON i.id = d.itinerary_id WHERE i.user_id = auth.uid())).

    Match policy names to RESEARCH.md (places_select_all, places_insert_service, places_update_service, itineraries_owner, itineraries_share_select, itinerary_days_owner, place_visits_owner).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| any authenticated user → itinerary tables | DB-layer RLS must prevent cross-user access |
| public share viewer → itineraries | is_public must gate read access server-side |
| request → middleware → session cookies | Session refresh handles auth tokens in cookies |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-05 | Elevation of Privilege | itinerary tables | high | mitigate | RLS owner policies scope every row to auth.uid(); itinerary_days/place_visits inherit ownership via subquery joins |
| T-01-06 | Information Disclosure | public share access | high | mitigate | itineraries_share_select policy requires is_public = true AND share_token; enforced at DB layer, not UI |
| T-01-07 | Tampering | places shared cache writes | medium | mitigate | places INSERT/UPDATE policies restrict writes to auth.role() = 'service_role' |
| T-01-08 | Spoofing | session cookies | medium | mitigate | @supabase/ssr middleware refreshes session on every request; getUser() validates the user server-side |
| T-01-SC | Tampering | npm installs (@supabase/ssr) | high | accept | @supabase/ssr is the first-party Supabase package pre-approved in CLAUDE.md; no [ASSUMED]/[SUS] packages added |
</threat_model>
